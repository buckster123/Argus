## Architecture (One Paragraph)

Argus is a plugin-based sensor framework. Each sensor implements `SensePlugin` (async probe/read/stream/close). A `PluginRegistry` auto-discovers plugins at startup and probes them; only available sensors expose MCP tools. The MCP server (`argus/mcp_server.py`) uses stdio transport and dynamically registers tools per sensor. The FastAPI dashboard (`argus/dashboard.py`) serves REST + SSE on port 8080 with a dark HTML UI. All sensor I/O runs in `asyncio.to_thread()` to avoid blocking the event loop.

## Critical Conventions

1. **Plugin naming:** Module name = sensor name (e.g., `webcam.py` → `name="webcam"`). Tool names = `argus_{name}_{action}`.
2. **Probe must be fast:** < 1s timeout. Open camera briefly, then release. No lingering handles.
3. **Thread safety:** All hardware I/O goes through `asyncio.to_thread()`. Plugins own their locks.
4. **Graceful degradation:** Missing deps = plugin silently skipped during discovery. No import errors.
5. **Image format:** JPEG bytes in `SenseData.image`. Audio: WAV bytes in `SenseData.audio`.
6. **MCP ImageContent:** Base64-encoded JPEG string. `mimeType="image/jpeg"`.
7. **No blocking in probe():** Use `asyncio.to_thread()` if probe does I/O.

## Testing Strategy

- **Unit tests:** Mock hardware deps (cv2, mss, psutil, sounddevice). Test probe/read/encode cycles.
- **Integration tests:** Require actual hardware. Marked with `@pytest.mark.hardware` or `@pytest.mark.display`.
- **Fast test suite:** `make test-fast` skips hardware/display tests. CI uses this.
- **Coverage target:** ≥80% for v0.2.0.

## Adding a New Sensor (Template)

```python
# argus/senses/my_sensor.py
from argus.senses.base import SenseData, SensePlugin

class MySensorPlugin(SensePlugin):
    name = "my_sensor"
    capabilities = ["capture"]  # or "stream", "detect"

    async def probe(self) -> bool:
        # Check if hardware/library available
        return _HAS_DEPS

    async def read(self) -> SenseData:
        # Return SenseData with .data, .text, optionally .image/.audio
        pass

    def mcp_tools(self):
        return [{"name": "argus_my_sensor_read", ...}]
```

No registration needed — `PluginRegistry.discover()` finds it automatically.

## MCP Tool Return Format

`_sense_to_mcp()` returns a list in this priority:
1. `TextContent` with error (if any)
2. `TextContent` with human-readable `.text`
3. `ImageContent` with base64 JPEG (if `.image` present)
4. `TextContent` with base64 WAV note (if `.audio` present)
5. `TextContent` with JSON `.data`

## Environment Notes

- Headless servers: `screen` probe returns False if `$DISPLAY` unset. `webcam` may still work via v4l2.
- Audio: Requires ALSA/PulseAudio. `sounddevice` fails gracefully if no devices.
- Windows: mss works. OpenCV works. psutil works. sounddevice works.
- macOS: All universal sensors work. Screen capture includes menu bar.

## Integration Points

- **Hermes:** MCP server config points to `argus-mcp` script. Dashboard plugin in Phase 4.
- **CerebroCortex:** Can ingest sensor logs from `/api/{sensor}/read` endpoints.
- **SensorHead (Pi):** ESP32 serial co-processor plugin planned as bridge between Argus and SensorHead hardware.

## Common Pitfalls

- Don't cache `VideoCapture` across reads without a lock — camera contention crashes.
- Don't forget `await instance.close()` in registry cleanup.
- Don't import heavy deps at module level without `_HAS_X` guard — breaks discovery.
- `ImageContent.data` must be base64 string, not bytes.

## Decision Log

- **Why stdio MCP?** Simplest transport, works with Claude Code, Hermes, any MCP client.
- **Why JPEG not PNG?** Smaller over the wire. Quality 85 is the sweet spot.
- **Why not async-native OpenCV?** OpenCV is C++ and blocking. `to_thread()` is the pragmatic choice.
- **Why global config singleton?** MCP server and dashboard share config. Mutable but contained.
